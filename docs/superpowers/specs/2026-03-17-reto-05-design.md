# Reto 05 — Restaurant Inventory Automation
**Date:** 2026-03-17
**Status:** Approved

## Problem

Restaurant managers manually track inventory and sales across disconnected systems. Low stock goes unnoticed until a product runs out mid-service. Reordering from suppliers is manual, slow, and prone to being forgotten.

## Solution

A Make.com automation that connects Square POS (sales) with Google Sheets (inventory + suppliers) to detect low stock automatically, alert the manager, and send AI-generated reorder requests to suppliers via email and WhatsApp.

## Architecture

Two independent Make scenarios:

- **Scenario 1** — Triggered by each completed Square sale; updates inventory in real time
- **Scenario 2** — Scheduled every 6 hours; detects low stock, alerts manager, contacts suppliers

## Data Structure

### Google Sheets — Hoja 1: Inventario

| Column | Type | Description |
|--------|------|-------------|
| product_id | string | Unique product identifier |
| product_name | string | Human-readable product name |
| current_stock | number | Current stock level (updated per sale) |
| min_threshold | number | Minimum acceptable stock level |
| unit | string | Unit of measurement (kg, units, liters, etc.) |

### Google Sheets — Hoja 2: Proveedores

| Column | Type | Description |
|--------|------|-------------|
| product_id | string | Foreign key matching Inventario |
| product_name | string | Product name |
| supplier_name | string | Supplier company or contact name |
| supplier_email | string | Email address for formal orders |
| supplier_whatsapp | string | WhatsApp number (E.164 format) |
| reorder_quantity | string | Default quantity to reorder (e.g. "50 kg") |

## Scenario 1 — Inventory Update

**Trigger:** Square → Watch Orders (order status: completed)

**Modules:**
1. **Square: Watch Orders** — fires on each completed sale
2. **Iterator** — iterates over each line item in the order
3. **Google Sheets: Search Rows** — finds the matching row in Inventario by `product_id`
4. **Google Sheets: Update Row** — subtracts quantity sold from `current_stock`

**Outcome:** Inventario always reflects real stock after every sale.

## Scenario 2 — Low Stock Detection, Alert & Auto-Order

**Trigger:** Scheduled every 6 hours

**Modules:**
1. **Google Sheets: Search Rows** — retrieves all rows where `current_stock` ≤ `min_threshold`. If no rows are returned, Make produces zero bundles and the scenario ends naturally — no explicit exit module needed.
2. **Array Aggregator** — aggregates all low-stock product bundles into a single array so the manager alert can be sent as one consolidated email listing all affected products
3. **Email: Send an Email** — sends manager alert with a single consolidated list of all low-stock products (name, current stock, threshold, unit)
4. **Iterator** — iterates over each low-stock product for individual supplier outreach
5. **Google Sheets: Search Rows** — looks up supplier in Proveedores by `product_id`
6. **Anthropic Claude: Message** — generates a formal reorder email using the following prompt template:
   > "You are an assistant for a restaurant. Write a formal, professional email in Spanish to a supplier requesting a product reorder. The restaurant needs {{reorder_quantity}} of {{product_name}} as soon as possible due to low stock. The supplier's name is {{supplier_name}}. Write only the email body — no subject line. Keep it concise and polite."
7. **Email: Send an Email** — sends Claude-generated order to `supplier_email`
8. **WhatsApp Business Cloud: Send a Message** — sends a short notification to `supplier_whatsapp` (E.164 format, e.g. `+521234567890` — no `whatsapp:` prefix required, unlike Twilio) confirming a formal order was sent by email

**Outcome:** Manager receives one consolidated alert email. Each affected supplier receives an AI-drafted order email in Spanish and a WhatsApp confirmation.

Total: **9 modules across 2 scenarios** — exceeds the 4-module minimum.

## Error Handling

- If a `product_id` from a Square order is not found in Inventario, Make's native execution history captures the incomplete bundle automatically — no additional module needed. The scenario continues processing remaining line items.
- If a low-stock product has no matching supplier in Proveedores, skip supplier contact and include the product in the manager alert with a note: "No supplier configured"
- WhatsApp delivery failure does not block the email flow
