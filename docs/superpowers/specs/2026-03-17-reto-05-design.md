# Reto 05 — Restaurant Inventory Automation
**Date:** 2026-03-17
**Status:** Approved

## Problem

Restaurant managers manually track inventory and sales across disconnected systems. Low stock goes unnoticed until a product runs out mid-service. Reordering from suppliers is manual, slow, and prone to being forgotten.

## Solution

A Make.com automation that connects Square POS (sales) with Google Sheets (inventory + suppliers) to detect low stock automatically, alert the manager, and send AI-generated reorder requests to suppliers via email and WhatsApp.

## Architecture

Two independent Make scenarios:

- **Scenario 1** — Triggered by each completed Square sale; updates inventory in real time
- **Scenario 2** — Scheduled every 6 hours; detects low stock, alerts manager, contacts suppliers

## Data Structure

### Google Sheets — Hoja 1: Inventario

| Column | Type | Description |
|--------|------|-------------|
| product_id | string | Unique product identifier |
| product_name | string | Human-readable product name |
| current_stock | number | Current stock level (updated per sale) |
| min_threshold | number | Minimum acceptable stock level |
| unit | string | Unit of measurement (kg, units, liters, etc.) |

### Google Sheets — Hoja 2: Proveedores

| Column | Type | Description |
|--------|------|-------------|
| product_id | string | Foreign key matching Inventario |
| product_name | string | Product name |
| supplier_name | string | Supplier company or contact name |
| supplier_email | string | Email address for formal orders |
| supplier_whatsapp | string | WhatsApp number (E.164 format) |
| reorder_quantity | string | Default quantity to reorder (e.g. "50 kg") |

## Scenario 1 — Inventory Update

**Trigger:** Square → Watch Orders (order status: completed)

**Modules:**
1. **Square: Watch Orders** — fires on each completed sale
2. **Iterator** — iterates over each line item in the order
3. **Google Sheets: Search Rows** — finds the matching row in Inventario by `product_id`
4. **Google Sheets: Update Row** — subtracts quantity sold from `current_stock`

**Outcome:** Inventario always reflects real stock after every sale.

## Scenario 2 — Low Stock Detection, Alert & Auto-Order

**Trigger:** Scheduled every 6 hours

**Modules:**
1. **Google Sheets: Get All Rows** — retrieves all inventory rows. If the sheet is empty, Make produces zero bundles and the scenario ends naturally.
2. **Filter** — passes only bundles where `current_stock` ≤ `min_threshold`. Make's standalone Filter module supports comparing two mapped fields from the same bundle. Using a separate Filter (rather than Search Rows' built-in filter) is required because Search Rows only accepts literal values as comparators, not dynamic field references.
3. **Text Aggregator** — builds a single plain-text list of all low-stock products (one line per product) for the consolidated manager email.
4. **Email: Send an Email** — sends manager alert with the aggregated list. Make does not support Handlebars or loop syntax in email fields; the Text Aggregator is the correct pattern for building multi-item email bodies.
5. **Google Sheets: Get All Rows** — second pass to re-fetch all inventory rows for per-product supplier outreach. Necessary because the Text Aggregator output is a string, not structured data that can be iterated.
6. **Filter** — same condition as module 2: passes only low-stock products.
7. **Google Sheets: Search Rows** — looks up supplier in Proveedores by `product_id` (from module 6). If no supplier is found, Make produces zero bundles and modules 8–10 are skipped for that product.
8. **Anthropic Claude: Message** — generates a formal reorder email in Spanish. Prompt uses `{{6.product_name}}` (Inventario), `{{7.supplier_name}}`, and `{{7.reorder_quantity}}` (Proveedores).
9. **Email: Send an Email** — sends Claude-generated order to `supplier_email` (from module 7).
10. **WhatsApp Business Cloud: Send a Message** — sends a short notification to `supplier_whatsapp` (E.164 format, e.g. `+521234567890` — no `whatsapp:` prefix required, unlike Twilio).

**Outcome:** Manager receives one consolidated alert email. Each affected supplier with a configured entry in Proveedores receives an AI-drafted order email in Spanish and a WhatsApp confirmation.

## Error Handling

- If a `product_id` from a Square order is not found in Inventario, Make's native execution history captures the incomplete bundle automatically — no additional module needed. The scenario continues processing remaining line items.
- If a low-stock product has no matching supplier in Proveedores, skip supplier contact and include the product in the manager alert with a note: "No supplier configured"
- WhatsApp delivery failure does not block the email flow

## Reto Requirements

- **Connects two tools that don't normally talk:** Square POS ↔ Google Sheets
- **Solves a common problem:** universal to any restaurant or business with physical inventory
- **Minimum 4 modules:** 14 modules across 2 scenarios
- **Useful beyond the creator:** any restaurant owner can clone and configure this automation
